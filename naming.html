5. Method Naming

A.
General
1.
Method Library

The full title is the Central Council Method Library. This is the definitive record of methods that have been rung and named, which is maintained and made available by the Central Council.

2.
Method Title

Methods recorded in the Method Library have a unique tripartite Method Title consisting of: Name, Class Descriptor, Stage Name.

Further explanation: The Class Descriptor may contain more than one term, or in some cases, notably Principles and historic methods like Grandsire, no term, i.e. the Class Descriptor is null, as defined in the next section.

3.
Rotation

A Static Method whose Changes are the same as another Static Method, except that the Changes, when considered as a cycle, start at a different point in the cycle.

Example: The Method Plain Bob Minimus has the Changes x14x14x14x12. The Static Method with the Changes x14x12x14x14 is a Rotation of Plain Bob Minimus.

Further explanation: More fully, this term would be 'Method Rotation' to distinguish it from other uses of rotation in method ringing, such as a rotated Block or a rotated Composition (Compositions are covered in Section G below).

Technical comments: When using a Static Method's sequence of Changes to produce Rows, the Changes may be used starting and finishing at any point in the sequence. For this reason, a Rotation of an existing Static Method is not usually separately named, because the same Changes can be produced by starting and finishing the existing Static Method at a different point in its sequence of Changes.

4.

Methods recorded in the Method Library may not have names that conflict with the criteria related to method extension (see Section 8) or symmetry relationships (see Section 5.C).

5.

Methods recorded in the Method Library that have in the past been known by a different name and/or title will have this recorded, where details are known.

B.
Class Descriptors
1.
Class descriptor

The Class Descriptor is formed by concatenating up to four terms as follows, and in the order shown:
a) Add ’Jump’ if the Method has this Class;
b) Add ‘Differential’ if the Method has this Class;
c) Add ’Little’ if the Method has this Class;
d) Add one of the following if it applies: ‘Place’, ‘Bob’, ‘Treble Bob’, ‘Surprise’, ‘Delight’, ‘Treble Place’, ‘Alliance’;
e) Otherwise leave blank.

C.
Naming Constraints
1.

The following constraints apply to a new Method to be added to the Method Library.

2.

A new Method may not use the same name as a Method recorded in the Method Library if that would result in its Method Title being the same.

3.

The name given to a new Method that is recorded in the Method Library may not include Class Descriptor term(s) such that the resulting Method Title is ambiguous as to the Class of the Method.

Example: Naming a Bob Method “Quavo Treble” would give a misleading Method Title, e.g. “Quavo Treble Bob Major”. It's unclear from the Method Title whether the Method is a Bob Method or a Treble Bob Method.

4.

Rotations of a Static Method are considered to be the same Method and will normally only be recorded under one name in the Method Library. However, this is not mandated as there are a few cases where it is beneficial to record an alternative name for a Rotation.

Further explanation: When using a Static Method's sequence of Changes to produce Rows, the Changes may be used starting and finishing at any point in the sequence. For this reason, a Rotation of an existing Static Method is not usually separately named, because the same Changes can be produced by starting and finishing the existing Static Method at a different point in its sequence of Changes.

An example where separate naming of Rotations is beneficial is for twin hunt methods like Grandsire Triples. When rotated so that the second hunt bell is the treble, and rung with Grandsire calls affecting the first hunt bell rather than the second, it seems like a different method. This rotation has been rung many times under the name New Grandsire (7.1.7.1.7.1.7.1.7.1.7.1.3.1) and has also been spliced with Grandsire. Separate inclusion of New Grandsire in the Method Library is therefore considered beneficial.

5.

The sequence of Changes of a Static Method recorded in the Method Library is normally not divisible into two or more equal parts.

Further explanation: A Static Method might have the sequence of Changes 34x34.14.12.14.34x34.14.12.14, representing one Plain Lead. This Method would have two Plain Leads in its Plain Course. However, since this sequence of Changes is divisible into two equal parts (each with place notation 34x34.14.12.14), it would normally be recorded in the Method Library as the sequence 34x34.14.12.14, thus a Method with four Plain Leads in its Plain Course. The longer place notation shown above can be obtained simply by using two Plain Leads of the shorter Method.

However there can be reasons to record a Static Method in the Method Library that has a divisible sequence of Changes. An example is Plain Hunt. On 6 bells the place notation is x16x16x16x16x16x16. This is recorded in the Method Library as the Method called Original with place notation x16. But since Plain Hunt is widely used and recognized as a Method Name, it is considered beneficial for Plain Hunt to be recorded as a Method in the Method Library, in addition to Original.

6.

The prefixes ‘Reverse’, ‘Single’ and ‘Double may only be used to begin a Method Name if the relevant symmetry relationship is satisfied (see below), in which case their use is optional.

7.

A Method Name may only begin with ‘Reverse’ if:
a) Another Method in the Method Library has the same Method Title (after excluding ‘Reverse’ in the new Method); and
b) That other Method's Changes are the Reverse of the new Method's -- that is, the Places within each Change are inverted.

Example: St Simon's Bob Doubles has the place notation 5.1.5.3.5.3.5.1.5.125.
If each place notation is inverted, the result is 1.5.1.3.1.3.1.5.1.145. This method (usually rotated to be rung as 3.1.5.1.145.1.5.1.3.1) is Reverse St Simon's Bob Doubles.

8.

A Method Name may only begin with ‘Double’ if the following conditions are both met:
a) It is a Double Method -- that is, when the Places of each Change are inverted, the same Method results;
b) If there is an existing Method in the Method Library that has the same Method Title (after excluding ‘Double’ in the new Method, or after excluding ‘Double’ in the new Method and excluding 'Single' in the existing Method), then both Methods have the same place notation above the Hunt Bell (or two or more coursing Hunt Bells).

Example: Double Court Bob Minor has the place notation x14x36x16x36x14x16.
If these changes are inverted, the result is x36x14x16x14x36x16. This can be rotated to give the same Changes as above.
It is therefore a Double method.
There is also a Method called Single Court Bob Minor in the Method Library. It's place notation is x14x16x16x16x14x16.
Because the places above the hunt bell in Single Court Bob Minor are the same as in Double Court Bob Minor, Double may be used.

9.

A Method Name may only begin with ‘Single’ if the following conditions are both met:
a) It is not a Double Method as defined in 8 a) above;
b) If there is an existing Method in the Method Library that has the same Method Title (after excluding ‘Single’ in the new Method, or after excluding 'Single' in the new Method and excluding ‘Double’ in the existing Method), then both Methods have the same place notation above the Hunt Bell (or two or more coursing Hunt Bells).

Example: See the example of Single Court Bob Minor and Double Court Bob Minor in 7. above.

10.

If one Extension Process has been used to relate two or more Methods that share a Name in the Method Library then:
a) A new Method at a different Stage that shares the same relationship shall use the same Name; and
b) A new Method at a different Stage that does not share the same relationship shall not use the same Name, even if it shares a relationship with one of the Methods based on a different Extension Process.

Example: Yorkshire Surprise Royal is related to Yorkshire Surprise Major by an Extension Process, hence these Methods may share a Name.
Southwark Surprise Maximus is also related to Yorkshire Surprise Major by a different Extension Process. Since the extension process for the name Yorkshire has already been established by the naming of Yorkshire Surprise Royal, Southwark could not have been named Yorkshire Surprise Maximus.

11.

If a new method is related to more than one method in the Method Library by different Extension Processes, and none of the existing Methods are related by an Extension Process to any other Method in the Library, then any of the valid Extension Processes may be used to name the new Method, but consideration should be given to:
a) The number of stages at which the Extension Process would create valid Methods; and
b) The number of desirable characteristics that are retained, as described in Section 8.B.

D.
Variations
1.

A Variation is a named combination of a Method and a Call or Calls.

2.

The Central Council's Variations and Calls Libraries only record Doubles Variations and Calls.

Further explanation: At Stages other than Doubles, Method Names are independent of the Call or Calls used.

3.

Each Variation recorded in the Central Council's Variations Library:
a) Is capable of producing a single Extent;
b) Has the same Class as the underlying Method on which it is based;
c) Has a Name that is unique across both Methods and Variations with the same Class Descriptor (including a null Class Descriptor); and
d) Satisfies the same naming constraints as the underlying Method (see Section 5.C).

4.

Variations are incorporated into Compositions in the same way as Methods.

E.
Right to Name
1.

An unnamed Method may be named by the band that first rings it in a Performance that is reported by the Ringing World, providing:
a) It either comprises at least 1250 Changes, or
b) It contains an Extent of the Method; and
c) The name does not conflict with other requirements in this framework.

Further explanation: "Reported by the Ringing World" includes in print and/or on BellBoard.

2.

An unnamed Variation may be named by the band that first rings it in a Performance that is reported by the Ringing World, providing:
a) It contains an Extent of the Variation; and
b) The name does not conflict with other requirements in this framework.

3.

The Central Council reserves the right to change the name proposed for a new Method or Variation, or to leave it unnamed, if it considers this necessary.

Further explanation: This could include considering that the proposed naming is misleading or offensive, or that the naming of an Extension or Contraction is inappropriate.

F.
Exceptions
1.

By convention the Class Descriptor is omitted from the Method Title of the following Methods, although they are classified as Bob Methods:
a) Grandsire,
b) Double Grandsire,
c) Reverse Grandsire,
d) Little Grandsire,
e) Union,
f) Double Union,
g) Reverse Union.

2.

The above Method Names may not be reused for either new Bob Methods or new Methods with no Class Descriptor.

3.

The Method known as Little Bob has no Method Name since Little Bob is its Class Descriptor. It will continue to be known as Little Bob.

4.

The Method Name ‘Plain’, which is the most likely name for Little Bob if it had one, may not be used for any new Little Bob Method.
